## Definitions (reconstruction method)
Per case, from `framework_guided.history`:
- **K=1 (autonomous)** = outcome if we STOP after the first pass (no oracle). Turn-1
  `diagnostic`: has `is_correct` → the system committed a number (`is_correct` gives
  correct/silent-wrong); no `is_correct` (status error/refusal/needs_clarification) → the
  system **deferred/abstained** (asked instead of guessing).
- **K=3 (clarify)** = `final_is_correct` / `final_status` (up to 3 oracle-assisted turns).
- **Silent-wrong** = committed a numeric answer that is wrong. **Abstain** = produced no
  committed answer (F/M/N/E/I gate). **Coverage** = fraction answered. **Selective-acc** =
  accuracy among answered.
- Note: this "recovery" (31) is defined as *not-correct@turn-1 → correct@final*; the summary's
  `recovery_count=35` uses a looser "turn-1 recorded an error" definition. Use the reconstructed
  31 for the honest recovery number.

## What this shows (and how it answers the two reviewer attacks)

**Attack: "K=3 93.6% is recoverability with an intent-oracle, not autonomous accuracy."**
Correct — and now quantified. **Autonomous K=1 = 81.2%**; the 12.4-pt lift to 93.6% is
oracle-assisted recovery of 31 cases at avg 2.1 rounds. Report K=1 as the headline autonomous
number; label 93.6% as *clarification-assisted recoverability* with the cost column visible.

**The real, attack-resistant win is failure behavior, not accuracy:**
- Autonomous VerifiQuant (81.2%) ≈ CoT-**with-oracle** (82.0%) on accuracy — a wash. **Do not
  race on accuracy.**
- But **silent-wrong**: VQ **2.8% (K=1) / 3.2% (K=3)** vs CoT **22.4% / 18.0%** — a **~6–8×
  reduction**. This is the thesis ("predictable, verifiable failure behavior"), and it holds at
  every setting.
- VQ **abstains in a calibrated way** (16% at K=1, selective-acc 0.967); CoT never abstains and
  pays for it in silent-wrongs.

**Caveat to pre-empt (coverage mismatch):** VQ's accuracy is at <100% coverage (it abstains),
CoT's is at 100%. Point accuracy is therefore not directly comparable — the honest comparison is
the **risk–coverage / selective-prediction** view: at equal coverage VQ's selective accuracy
(0.967) dominates CoT (0.78–0.82). Add a risk–coverage curve to the paper.

## Risk–coverage (answers the coverage-mismatch objection)

Figure: `docs/figures/risk_coverage_250q.png`. CoT has no confidence signal to threshold, so we
plot its **perfect-abstention upper bound** (best case if it could rank and reject its own wrong
answers first) and overlay VerifiQuant's **real** operating points (K=1→3).

VerifiQuant real operating points (history-truncated per K):
- K=1: coverage 84.0%, selective-acc **96.7%**
- K=2: coverage 96.0%, selective-acc 96.2%
- K=3: coverage 96.8%, selective-acc 96.3%

Matched-coverage comparison **at 84% coverage** (VQ's autonomous operating point):
- CoT single-shot perfect-abstention **upper bound** = 92.4%
- CoT+oracle perfect-abstention **upper bound** = 97.6%
- **VerifiQuant real = 96.7%**

Reading:
- **Autonomous (fair K=1 comparison):** VerifiQuant's *real* selective policy (96.7%) beats CoT
  single-shot's *theoretical best* (92.4%) by +4.3 pts — a real deployed abstention policy beats
  what CoT could do only with a perfect (unavailable) rejector.
- **At high coverage / oracle-assisted (K=3):** VQ (96.3% @ 96.8%) vs CoT+oracle real point
  (82.0% @ 100%) or its best case at 96.8% coverage (≈84.7%) — VQ dominates by ~12 pts.
- **Honest caveat:** at 84% coverage the CoT+oracle *upper bound* (97.6%) is ~1 pt above VQ-K=1
  (96.7%) — but that pits VQ *without* the oracle against CoT *with* the oracle's theoretical
  best, which favours CoT; even so VQ is within 1 pt. Don't hide this.

Note: figure K=3 point (history-truncated, 96.3%/93.2% acc) differs by <0.5 pt from the
authoritative final (93.6%); the main table's final numbers are canonical.
